# Growth Loops

Sustainable growth is a system. Incentive-only growth is a loan you eventually repay.

## The Two Types of Growth

| Type | Source | Lifecycle |
|---|---|---|
| Mercenary | Token emissions, APY incentives | Leaves when incentives drop |
| Sticky | Product value, community, switching cost | Stays and compounds |

Most protocols get mercenary growth first and confuse it for sticky growth. The goal: convert mercs to believers before incentives decay.

## Retention Framework — The First 30 Days

The first 30 days determine long-term retention. Most protocols have no plan for this.

### Week 1: Activation

Get users to their first meaningful action.

- [ ] Onboarding flow: fewer than 3 steps to first value (not a 10-step wallet setup)
- [ ] Tutorial or guided first transaction (show them what the protocol does, do not tell)
- [ ] First-action reward: small, immediate, on-chain (not "check back in 7 days")

### Week 2 to 3: Habit Formation

Get users to return without a push notification.

- [ ] Daily or weekly protocol interaction that generates visible on-chain output
- [ ] Streaks, progress, or on-chain achievements (Solana programs can emit this cheaply)
- [ ] Community events tied to protocol usage, not price

### Week 4: Loyalty Identification

Find your top 10% users and treat them differently.

- [ ] On-chain query: top 100 users by interaction count, volume, or protocol-specific metric
- [ ] Direct outreach to top users: ask what is working, what is not
- [ ] Power user role in Discord with meaningful benefits (early access, governance weight)

## TVL Sustainability

**Mercenary TVL equation:**
```
TVL = f(APY emissions)
Cut APY → TVL drops
```

**Sticky TVL equation:**
```
TVL = f(product utility, switching cost, community, perceived safety)
APY is secondary
```

### Moving from mercenary to sticky TVL

**Step 1: Measure retention cohorts**

```sql
-- Dune Analytics: weekly retention cohort (Solana)
WITH first_use AS (
  SELECT tx_signer, MIN(DATE_TRUNC('week', block_time)) as cohort_week
  FROM solana.transactions
  WHERE account_keys LIKE '%YOUR_PROGRAM_ADDRESS%'
  GROUP BY tx_signer
),
activity AS (
  SELECT tx_signer, DATE_TRUNC('week', block_time) as active_week
  FROM solana.transactions
  WHERE account_keys LIKE '%YOUR_PROGRAM_ADDRESS%'
  GROUP BY tx_signer, active_week
)
SELECT
  f.cohort_week,
  DATEDIFF('week', f.cohort_week, a.active_week) as weeks_since_start,
  COUNT(DISTINCT a.tx_signer) as retained_users
FROM first_use f
JOIN activity a ON f.tx_signer = a.tx_signer
GROUP BY 1, 2
ORDER BY 1, 2
```

Target: Week-4 retention above 30% of Week-1 cohort.

**Step 2: Build fee-based yield (real yield)**

Real yield = protocol revenue distributed to stakers/LPs. Not emissions.

```
Real Yield APY = (annualized_protocol_fees × distributed_fraction) / staked_token_value
```

When real yield is above 0 and growing: you have a business, not a token.

**Step 3: Design switching costs**

Switching costs that are product-natural, not lock-up exploitative:
- Reputation or score built on-chain over time
- Preferential fee tier for long-tenure users
- Governance weight proportional to protocol age and usage
- Social graph: users' connections are within the protocol

## Incentive Program Design

### Liquidity Mining

Use when: early stage, need to bootstrap LP depth.

Rules for sustainable programs:
- Fixed duration (6 to 12 weeks max per epoch)
- Declining emission schedule (not flat)
- Require protocol usage, not just LP deposits
- Tie emissions to fee generation, not TVL

Good: "Earn TOKEN proportional to fees you generate for LPs"
Bad: "Deposit and earn 200% APY indefinitely"

### Points and Pre-Token Programs

Use when: protocol is live but token has not launched.

Rules:
- Clear conversion methodology published upfront
- Anti-farming: actions require real usage (multi-step, capital at risk)
- Time-weighted: early users weighted higher (rewards commitment)

### Referral Programs

Use when: product has strong activation (above 40% week-1 retention).

Rules:
- Reward the referrer for referred user's activity, not sign-up
- Cap per-user referral rewards (prevents referral ring attacks)
- On-chain referral tracking (not off-chain link tracking which is gameable)

## Growth Metrics That Actually Matter

Track these weekly, not follower count:

| Metric | Formula | Target |
|---|---|---|
| DAU/MAU ratio | Daily active / Monthly active | Above 0.2 (sticky), below 0.05 (mercenary) |
| Protocol revenue | Fees generated by protocol | Growing week-over-week |
| Real yield APY | Protocol fees / staked value | Above 5% annualized = healthy |
| Retention D30 | Users active on day 30 / day 1 | Above 25% |
| Organic volume % | Volume not incentivized | Growing fraction of total |
| TVL per user | Total TVL / active wallets | Growing = users deploying more capital |

## The Growth Trap to Avoid

Token price is not a growth metric.

Protocols that optimize for token price over product usage always end the same way: token dumps, community disappears, TVL vanishes, team loses credibility.

The compound effect runs in reverse too. Product utility leads to organic usage, which leads to token demand. Build the first two. The third follows.
